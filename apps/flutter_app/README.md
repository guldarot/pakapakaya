# PakaPakaya Flutter

This directory contains a mobile-first Flutter rebuild foundation for PakaPakaya.

What is implemented:
- Feature-first Flutter project structure
- Canonical domain models and enums aligned to the docs-first product plan
- Request/response DTO scaffolding for the new backend contract
- Mock backend to exercise key buyer/vendor flows without a live API
- Core app routes for auth, discovery, vendor shop, order detail, chat, vendor center, and admin
- OpenAPI contract draft in `../../packages/contracts/openapi.yaml`
- Domain model notes in `docs/domain_model.md`

What is intentionally still pending:
- Real Flutter CLI bootstrap and dependency resolution in this environment
- Generated model code from `freezed` / `json_serializable`
- Production backend implementation
- Storage uploads, realtime socket integration, OTP provider integration, and full moderation flows

Recommended next steps:
1. Run `flutter pub get`
2. Run `flutter create .` only if you want platform folders generated by the Flutter CLI
3. Replace the mock backend with real repositories generated from the OpenAPI contract
4. Add PostgreSQL/PostGIS backend services and signed upload endpoints

Backend toggle:
- Mock mode is on by default.
- To point the app at the backend scaffold, run:

```powershell
flutter run -d chrome --dart-define=USE_MOCK_BACKEND=false --dart-define=API_BASE_URL=http://localhost:4000
```
